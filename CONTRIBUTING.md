# Contributing to Synapse Analysis

Welcome! This repository is maintained by a multi-lab consortium focused on synaptic plasticity. We welcome contributions that improve the analysis tools.

## Code Style

- Follow PEP 8 for Python code.
- Use descriptive variable names.
- Include docstrings for all public functions and classes.
- Write unit tests for new functionality when possible.

## Pull Request Process

1. Fork the repository and create a branch for your feature.
2. Ensure your code passes any existing tests.
3. Update documentation if needed.
4. Open a pull request with a clear description of changes.
5. Request review from the lead maintainer (see CODEOWNERS).

## Issue Reporting

Please open an issue to discuss major changes or report bugs. Use the issue template if available.
